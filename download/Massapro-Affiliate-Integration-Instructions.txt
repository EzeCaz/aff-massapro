QUESTION 6: WHEN SHOULD trackPurchase() FIRE? (CRITICAL)
----------------------------------------------------------
  Since the /all page redirects users to ClickBank
  (aireceptio.pay.clickbank.net), the user LEAVES the site after clicking
  "Buy Now". We cannot fire trackPurchase() client-side because:

    (a) The user is redirected to ClickBank and may not return to the site
    (b) We don't know at click time whether the user will complete payment
    (c) Firing trackPurchase() on the button click would count ALL clicks
        as purchases, inflating conversion data and incorrectly awarding
        commissions for users who never actually paid

  THEREFORE, we use OPTION A:

    trackCart()    →  FIRES ON "BUY NOW" CLICK (client-side)
                      This records the user's INTENT to purchase (add to
                      cart / checkout initiated). It does NOT trigger
                      commissions. It captures the affiliate attribution
                      from the cookie while we still have the user on site.

    trackPurchase()  →  FIRES SERVER-SIDE VIA CLICKBANK INSTANT
                         NOTIFICATION (postback)
                         ClickBank sends a server-to-server POST when a
                         payment is confirmed. This is the ONLY accurate
                         place to fire trackPurchase() because it confirms
                         the user actually paid. This triggers the
                         commission engine.

  DO NOT fire trackPurchase() on the "Buy Now" button click. That would
  treat every click as a completed purchase, which is inaccurate and would
  award commissions for users who abandon the ClickBank checkout.

  VERDICT: trackCart() on click, trackPurchase() via ClickBank postback.

---------------------------------------------------------------
  STEP 1: EMBED THE TRACKER SCRIPT ON YOUR WEBSITE
---------------------------------------------------------------

Add these two <script> tags to EVERY page of receptionist.massapro.com,
ideally just before the closing </body> tag (or in your site-wide layout):

  <!-- MassaPro Affiliate Tracker v3.0 -->
  <script src="https://aff-massapro.space-z.ai/massapro-affiliate-tracker.js"></script>
  <script>
    MassaProAffiliate.config({
      dashboardUrl: 'https://aff-massapro.space-z.ai'
    });
  </script>

WHAT THIS DOES:
  - When a visitor arrives with ?affid=XXX in the URL, the script:
    (a) Saves the affiliate ID in a 30-day cookie (massapro_affiliate)
    (b) Saves first-touch UTM parameters in a separate 30-day cookie
       (massapro_ft) — these are NEVER overwritten
    (c) Creates a session cookie (massapro_session, 1 day)
    (d) Fires a click tracking event to /api/track/click
    (e) Saves "landing" as the first funnel step

  - On subsequent page views (without ?affid=), the script:
    (a) Reads the existing affiliate cookie
    (b) Updates last-touch UTM parameters if new UTM params are present
    (c) Does NOT overwrite first-touch UTMs

COOKIE DETAILS:
  - massapro_affiliate: 30-day expiry, SameSite=Lax, contains:
      { affid, utm_source, utm_medium, utm_campaign, utm_content,
        utm_term, ft_utm_source, ft_utm_medium, ft_utm_campaign,
        ft_utm_content, ft_utm_term, landing_page, timestamp }
  - massapro_ft: 30-day expiry, first-touch UTMs (set once, never updated)
  - massapro_session: 1-day expiry, session tracking ID
  - massapro_funnel: 30-day expiry, funnel step progress


---------------------------------------------------------------
  STEP 2: TRACK BUTTON CLICK EVENTS
---------------------------------------------------------------

Add MassaProAffiliate.trackEvent(eventId) calls to your CTA buttons.
This fires TWO tracking requests:
  (1) A 1x1 pixel GET to /api/track/event — for event analytics
  (2) A POST to /api/track/click with event_type='button_click' — for
      click analytics with full UTM attribution

Use trackEvent() for informational CTAs that do NOT redirect to ClickBank:

  <!-- Hero CTA -->
  <button onclick="MassaProAffiliate.trackEvent('btn_hero_demo')">
    Book a Demo
  </button>

  <!-- Generic CTA -->
  <button onclick="MassaProAffiliate.trackEvent('btn_get_now')">
    Get Started Now
  </button>

  <!-- Navigation Links -->
  <a href="/pricing"
     onclick="MassaProAffiliate.trackEvent('nav_pricing')">
    View Pricing
  </a>

EVENT ID NAMING CONVENTION:
  Use a consistent prefix to categorize events:
    btn_      — Button clicks (btn_hero_demo, btn_get_now)
    nav_      — Navigation clicks (nav_pricing, nav_features)
    form_     — Form interactions (form_started, form_submitted)
    video_    — Video interactions (video_play, video_complete)
    scroll_   — Scroll milestones (scroll_pricing, scroll_testimonials)

NOTE: For the "Buy Now" pricing buttons that redirect to ClickBank,
use trackCart() instead of trackEvent() — see Step 3 below.

NOTE: If no affiliate cookie exists (i.e., the visitor was NOT referred
by an affiliate), trackEvent() will still fire the event pixel but will
skip the click POST (since there's no affiliate to attribute the click to).
The event is still recorded in the AffiliateEvent table for analytics.


---------------------------------------------------------------
  STEP 3: TRACK "BUY NOW" CLICKS AS ADD-TO-CART (CLICKBANK)
---------------------------------------------------------------

IMPORTANT: This is the most critical step. When a user clicks a "Buy Now"
button on the pricing section, they are redirected to ClickBank. This is
an "add to cart" or "checkout initiated" event — NOT a purchase. The user
has not yet paid. We use trackCart() here, NOT trackPurchase().

WHY trackCart() AND NOT trackPurchase():
  - The user is leaving the site to ClickBank
  - They may abandon the ClickBank checkout without paying
  - Firing trackPurchase() here would count every click as a purchase,
    awarding commissions for users who never actually paid
  - trackCart() records the INTENT to purchase without triggering
    commissions — that is the correct behavior

For each "Buy Now" button, add BOTH trackEvent() AND trackCart():

  <!-- Basic Plan Buy Button -->
  <a href="https://aireceptio.pay.clickbank.net/?plan=basic"
     onclick="MassaProAffiliate.trackEvent('btn_buy_basic');
              MassaProAffiliate.trackCart({
                plan_type: 'Basic',
                quantity: 1,
                cart_value: 29
              });">
    Buy Now — Basic
  </a>

  <!-- Professional Plan Buy Button -->
  <a href="https://aireceptio.pay.clickbank.net/?plan=professional"
     onclick="MassaProAffiliate.trackEvent('btn_buy_professional');
              MassaProAffiliate.trackCart({
                plan_type: 'Professional',
                quantity: 1,
                cart_value: 99
              });">
    Buy Now — Professional
  </a>

  <!-- Enterprise Plan Buy Button -->
  <a href="https://aireceptio.pay.clickbank.net/?plan=enterprise"
     onclick="MassaProAffiliate.trackEvent('btn_buy_enterprise');
              MassaProAffiliate.trackCart({
                plan_type: 'Enterprise',
                quantity: 1,
                cart_value: 199
              });">
    Buy Now — Enterprise
  </a>

WHY BOTH trackEvent() AND trackCart():
  - trackEvent('btn_buy_basic')  — Records the BUTTON CLICK for event
    analytics (which button was clicked, UTM attribution). This goes into
    the Click table and AffiliateEvent table.
  - trackCart({...})             — Records the ADD-TO-CART / CHECKOUT
    INITIATED event for the e-commerce funnel. This goes into the
    PurchaseEvent table and saves the funnel step. It does NOT trigger
    commissions.

WHAT trackCart() DOES:
  - Saves "add_to_cart" as a funnel step in the cookie
  - Fires an event pixel: add_to_cart_Professional (or whichever plan)
  - POSTs to /api/track/purchase with event_type: 'add_to_cart'
  - Records the event in the PurchaseEvent database table with full UTM
    attribution (first-touch + last-touch from cookies)
  - Does NOT trigger any commission (commissions only fire on purchase)

NOTE: Even if no affiliate cookie exists, the cart event is still
recorded (without affiliate attribution) for overall analytics.

CART VALUES — Update these to match your actual ClickBank prices:
  - Basic:        $29  (update to actual price)
  - Professional: $99  (update to actual price)
  - Enterprise:   $199 (update to actual price)


---------------------------------------------------------------
  STEP 4: TRACK LEAD FORM SUBMISSIONS
---------------------------------------------------------------

When a visitor submits a lead form (e.g., contact form, demo request),
call MassaProAffiliate.trackLead() to record the lead and attribute it
to the referring affiliate.

  MassaProAffiliate.trackLead({
    lead_name: 'John Smith',
    lead_email: 'john@company.com',
    lead_phone: '+1-555-0123',
    lead_company: 'Acme Corp',
    plan_type: 'Professional',        // "Basic", "Professional", "Enterprise"
    initial_status: 'Lead'            // "Lead", "Booked Call", "Paying Customer"
  })
  .then(function(result) {
    console.log('Lead tracked:', result.data);
  })
  .catch(function(err) {
    console.error('Lead tracking failed:', err);
  });

REQUIREMENTS:
  - lead_name is REQUIRED
  - An affiliate cookie MUST exist (visitor must have come via ?affid=XXX)
  - If no affiliate cookie exists, the call is rejected with a warning

COMMISSION TRIGGER:
  If initial_status is set to 'Paying Customer', the signup commission
  ($100 standard / $150 premium / custom) is immediately awarded to the
  affiliate. For other statuses, the commission is awarded later when the
  status is updated to 'Paying Customer' via /api/track/status.

FIRST-TOUCH + LAST-TOUCH UTM ATTRIBUTION:
  The trackLead() call automatically includes both first-touch and
  last-touch UTM parameters from the cookies. No manual UTM passing
  is needed — the tracker script handles it internally.


---------------------------------------------------------------
  STEP 5: TRACK COMPLETED PURCHASES VIA CLICKBANK POSTBACK
---------------------------------------------------------------

When ClickBank confirms a payment, it sends a server-to-server
Instant Notification (INS) to a URL you configure in your ClickBank
account. This is the ONLY accurate place to fire trackPurchase()
because it confirms the user actually paid.

DO NOT fire trackPurchase() on the client side. The user leaves your
site when they click "Buy Now" and goes to ClickBank. You have no way
to know client-side whether they completed payment.

CLICKBANK POSTBACK SETUP:

1. Log into your ClickBank account
2. Go to Account Settings → My Site → Instant Notification
3. Set the Notification URL to:

     https://aff-massapro.space-z.ai/api/track/clickbank

   (NOTE: This endpoint needs to be created — see the implementation
   section below.)

4. Select the notification types you want:
   - SALE          (required — fires when a purchase is completed)
   - BILL          (optional — fires for recurring billing)
   - REFUND        (optional — fires when a refund is issued)
   - CANCEL        (optional — fires when a subscription is cancelled)

5. Click "Save"

HOW THE CLICKBANK POSTBACK WORKS:

  ClickBank sends a POST to your notification URL with data like:

    cbreceipt      = "XXXXXXXX"          (ClickBank receipt ID — use as order_id)
    cbpop          = "XXXXXXXX"          (Verification hash)
    cbaffi         = "XXXXXXXX"          (ClickBank affiliate — NOT the same as
                                          your MassaPro affiliate)
    cbskin         = "XXXXX"             (Product ID)
    cbtrns         = "XXXXXXXX"          (Transaction ID)
    cbitem         = "XX"                (Item number)
    cbttrns        = "XXXXXXXX"          (Parent transaction for rebills)
    cbbfname       = "John"              (Buyer first name)
    cblname        = "Smith"             (Buyer last name)
    cbemail        = "john@example.com"  (Buyer email)
    cccountry      = "US"                (Country)
    cname          = "Product Name"      (What they bought)
    cprice         = "99.00"             (Price paid)
    ccur           = "USD"               (Currency)
    ctype          = "SALE"              (Transaction type)

  YOUR SERVER needs to:
    (a) Verify the cbpop hash (security — to confirm it's really from
        ClickBank and not a spoofed request)
    (b) Map the ClickBank item number to your MassaPro plan_type
        (Basic / Professional / Enterprise)
    (c) Look up the MassaPro affiliate using the buyer's email or a
        session identifier that was passed to ClickBank
    (d) Call the /api/track/purchase endpoint with the purchase data

CLICKBANK-TO-MASSAPRO AFFILIATE MAPPING — THE KEY CHALLENGE:

  ClickBank has its own affiliate system (cbaffi). This is SEPARATE from
  your MassaPro affiliate system. When ClickBank sends the postback, it
  includes cbaffi (the ClickBank affiliate) but NOT your MassaPro affid.

  To link a ClickBank purchase back to the correct MassaPro affiliate,
  you need to pass the MassaPro affid to ClickBank as metadata, then
  retrieve it in the postback. There are two approaches:

  APPROACH A: PASS affid IN THE CLICKBANK URL (RECOMMENDED)
  Append the MassaPro affid to the ClickBank checkout URL as a query
  parameter. ClickBank will include it in the postback as "cvendthru"
  (vendor through-parameter, max 100 chars).

  On the "Buy Now" link, change from:
    https://aireceptio.pay.clickbank.net/?plan=basic

  To:
    https://aireceptio.pay.clickbank.net/?plan=basic&cvendthru=MASSA001

  Where "MASSA001" is the affiliate's affid from the cookie. You can
  read it in JavaScript before the redirect:

    <a href="https://aireceptio.pay.clickbank.net/?plan=basic"
       onclick="var aff = MassaProAffiliate.getAttribution();
                if (aff && aff.affid) {
                  this.href = this.href + '&cvendthru=' + aff.affid;
                }
                MassaProAffiliate.trackEvent('btn_buy_basic');
                MassaProAffiliate.trackCart({ plan_type: 'Basic', quantity: 1, cart_value: 29 });">
      Buy Now — Basic
    </a>

  ClickBank will then include cvendthru=MASSA001 in the postback, and
  your server can use it as the affid for the trackPurchase() call.

  APPROACH B: MATCH BY BUYER EMAIL
  When the ClickBank postback arrives, use the buyer's email (cbemail)
  to search for an existing referral in the MassaPro dashboard:

    GET https://aff-massapro.space-z.ai/api/referrals?lead_email=john@example.com

  If a referral exists with that email and has an affiliate, you know
  which MassaPro affiliate to credit. This approach is less reliable
  because: (a) the user may not have submitted a lead form before
  purchasing, (b) the email on ClickBank may differ from the one they
  used on your lead form.

  RECOMMENDATION: Use Approach A (cvendthru) as the primary method,
  and Approach B (email matching) as a fallback.

CLICKBANK POSTBACK SERVER IMPLEMENTATION:

  A new endpoint needs to be created at:

    POST https://aff-massapro.space-z.ai/api/track/clickbank

  This endpoint will:
    1. Receive the ClickBank Instant Notification POST
    2. Verify the cbpop hash using your ClickBank secret key
    3. Parse the transaction type (SALE, BILL, REFUND, etc.)
    4. Extract the MassaPro affid from cvendthru
    5. Map the ClickBank item number to a MassaPro plan_type
    6. Call the existing /api/track/purchase logic with:
         event_type: 'purchase'
         order_id:   cbreceipt
         revenue:    cprice
         plan_type:  mapped from cbitem
         affid:      from cvendthru
         customer_email: cbemail
         customer_name:  cbbfname + " " + cblname
    7. Respond with 200 OK (ClickBank requires 200 to stop retrying)

  This endpoint will be implemented as part of the ClickBank integration
  phase. It requires your ClickBank secret key for cbpop verification.
  Once implemented, the full purchase-to-commission flow will work
  automatically:

    ClickBank SALE postback → /api/track/clickbank → purchase event →
    Referral created/updated → Commission awarded → Affiliate dashboard
    shows new conversion and earnings


---------------------------------------------------------------
  STEP 6: TRACK FUNNEL STEPS (OPTIONAL)
---------------------------------------------------------------

You can track custom funnel milestones at any point in the customer
journey. This helps visualize the conversion funnel in the dashboard.

  MassaProAffiliate.trackFunnelStep('view_pricing');
  MassaProAffiliate.trackFunnelStep('view_demo');
  MassaProAffiliate.trackFunnelStep('checkout_start');
  MassaProAffiliate.trackFunnelStep('payment_info_entered');

WHAT THIS DOES:
  - Saves the step name in the funnel cookie (massapro_funnel)
  - Fires an event pixel: funnel_view_pricing
  - Fires a click tracking event with event_type='funnel_step'

FUNNEL STEPS ARE ACCUMULATIVE:
  Each unique step name is added only once to the funnel cookie. If the
  same step is tracked multiple times, duplicates are ignored. The current
  funnel progress can be retrieved at any time:

  var progress = MassaProAffiliate.getFunnelProgress();
  // Returns: { steps: ["landing", "view_pricing", "add_to_cart", ...] }

AUTOMATIC FUNNEL STEPS:
  The tracker automatically records these steps:
    - "landing"        — When visitor first arrives with ?affid=
    - "lead_form"      — When trackLead() is called
    - "add_to_cart"    — When trackCart() is called (i.e., "Buy Now" click)
    - "purchase"       — When trackPurchase() is called (via ClickBank postback)


---------------------------------------------------------------
  STEP 7: UPDATE LEAD STATUS (SERVER-TO-SERVER)
---------------------------------------------------------------

When a lead's status changes (e.g., from "Lead" to "Booked Call" to
"Paying Customer"), call the /api/track/status endpoint from your server
or from the admin dashboard.

  PUT https://aff-massapro.space-z.ai/api/track/status
  Content-Type: application/json

  {
    "affid": "massa001",
    "referral_id": "cl_xxx",          // OR use lead_email + affid
    "lead_email": "john@company.com",
    "new_status": "Paying Customer",
    "plan_type": "Professional",
    "months_active": 1
  }

VALID STATUS VALUES:
  - "Lead"
  - "Booked Call"
  - "Paying Customer"
  - "Churned"

LEGACY STATUS MAPPING (automatically converted):
  - "Call Booked"       → "Booked Call"
  - "Active Subscriber" → "Paying Customer"

COMMISSION TRIGGERS ON STATUS UPDATE:
  - Transitioning to "Paying Customer" from any non-paying status:
    Awards the signup commission ($100/$150/custom) immediately.
  - Incrementing months_active for a "Paying Customer":
    Awards a recurring commission for each additional month
    (up to 12 months total).

ALTERNATIVE TO CLICKBANK POSTBACK:
  If ClickBank postback integration is delayed, you can also update
  lead status manually via the admin dashboard. When a ClickBank sale
  is confirmed, an admin can find the referral by email and update the
  status to "Paying Customer" — this triggers the commission. While
  manual, this ensures affiliates get credited even before the automated
  postback is implemented.


---------------------------------------------------------------
  STEP 8: INTEGRATE WITH META (FACEBOOK) PIXEL
---------------------------------------------------------------

If you fire Meta fbq('track', 'Purchase', ...) events on your site, note
that you should fire fbq('track', 'InitiateCheckout', ...) on the "Buy
Now" click (since the user is going to ClickBank, not completing a
purchase on your site). The Meta Purchase event should only fire if the
user returns to a thank-you page after ClickBank payment.

Example (fire on "Buy Now" click):

  // Meta Pixel — checkout initiated (NOT purchase)
  fbq('track', 'InitiateCheckout', {
    value: 99,
    currency: 'USD',
    content_name: 'Professional Plan'
  });

  // MassaPro Affiliate — add to cart / checkout initiated
  MassaProAffiliate.trackCart({
    plan_type: 'Professional',
    quantity: 1,
    cart_value: 99
  });

These are independent tracking systems — both should fire. The Meta
pixel does NOT communicate with the MassaPro affiliate system.

For Meta Purchase events, configure ClickBank to redirect to a thank-you
page on your site after payment, and fire fbq('track', 'Purchase', ...)
on that page.

================================================================================
  HOW UTM ATTRIBUTION WORKS
================================================================================

The MassaPro affiliate tracker uses a DUAL-TOUCH attribution model:

FIRST-TOUCH (massapro_ft cookie):
  - Captured ONCE when the visitor first arrives with ?affid=XXX
  - Never overwritten, even if the visitor returns via a different UTM source
  - Stored in the Referral and PurchaseEvent records as ft_utm_*
  - Used to answer: "Which campaign originally brought this customer?"

LAST-TOUCH (massapro_affiliate cookie):
  - Updated every time the visitor arrives with new UTM parameters
  - Stored in the Click records and in Referral/PurchaseEvent as lt_utm_*
  - Used to answer: "Which campaign drove the most recent conversion?"

EXAMPLE:
  Day 1:  Visitor clicks Google ad → ?affid=massa001&utm_source=google&utm_campaign=spring_sale
          First-touch: google / spring_sale
          Last-touch:  google / spring_sale

  Day 15: Visitor clicks Facebook retargeting ad → ?affid=massa001&utm_source=facebook&utm_campaign=retarget
          First-touch: google / spring_sale  (UNCHANGED)
          Last-touch:  facebook / retarget    (UPDATED)

  Day 20: Visitor clicks "Buy Now" → trackCart() fires with both UTMs
          PurchaseEvent (add_to_cart) records:
            ft_utm_source:   google
            ft_utm_campaign: spring_sale
            lt_utm_source:   facebook
            lt_utm_campaign: retarget

  Day 20: ClickBank confirms payment → trackPurchase() fires server-side
          PurchaseEvent (purchase) records same UTM attribution
          Commission awarded to affiliate massa001


================================================================================
  COMMISSION ENGINE DETAILS
================================================================================

SIGNUP COMMISSION (one-time):
  Triggered when a referral status becomes "Paying Customer", either:
    - Via trackLead() with initial_status: 'Paying Customer'
    - Via trackPurchase() with a valid affid and customer_email
      (fired server-side from ClickBank postback)
    - Via PUT /api/track/status with new_status: 'Paying Customer'
      (manual admin update or automated workflow)

  Amounts:
    Standard affiliate:  $100
    Premium affiliate:   $150
    Custom affiliate:    Per-affiliate override (customSignupComm field)

  IMPORTANT: trackCart() does NOT trigger commissions. Only trackPurchase()
  or a status update to "Paying Customer" triggers the commission engine.
  This ensures commissions are only awarded for confirmed payments.

RECURRING COMMISSION (monthly, up to 12 months):
  Triggered when months_active is incremented for a "Paying Customer"
  via PUT /api/track/status.

  Amounts per plan:
    Enterprise:     $100/month
    Professional:   $50/month
    Basic:          $10/month

  Custom affiliates may have different rates configured via:
    customEnterprise, customProfess, customBasic fields

COMMISSION LEDGER:
  Every commission is recorded in the CommissionLedger table with:
    - type: "signup", "recurring", or "adjustment"
    - amount: dollar amount
    - description: human-readable note
    - monthNumber: (for recurring) which month 1-12
    - referralId: link to the Referral record

AFFILIATE BALANCE TRACKING:
  - totalEarnings: sum of all commissions ever earned
  - approvedBalance: commissions available for payout
  - paidBalance: total commissions already paid out

================================================================================
  TROUBLESHOOTING
================================================================================

PROBLEM: trackEvent() calls seem to do nothing
  CAUSE:  The tracker script was not loaded or config() was not called.
  FIX:    Make sure both <script> tags are present and the dashboardUrl
          is set to 'https://aff-massapro.space-z.ai'.

PROBLEM: No affiliate data is being tracked
  CAUSE:  Visitor did not arrive via an ?affid= link, so no cookie was set.
  FIX:    Always test with an affiliate link that includes ?affid=XXX.

PROBLEM: Leads are not being attributed to affiliates
  CAUSE:  The affiliate cookie expired (30-day window) or was never set.
  FIX:    Ensure visitors arrive via affiliate links. Check that the
          massapro_affiliate cookie exists in DevTools.

PROBLEM: "Buy Now" click doesn't pass affid to ClickBank
  CAUSE:  The onclick handler isn't modifying the href before the
          redirect, or the affiliate cookie doesn't exist.
  FIX:    Check that the onclick runs before the href redirect. Make
          sure MassaProAffiliate.getAttribution() returns a valid affid.
          Test with a ?affid=XXX link first.

PROBLEM: Cart events show but no commissions are created
  CAUSE:  This is CORRECT behavior. trackCart() does NOT trigger
          commissions. Only trackPurchase() (via ClickBank postback)
          or a status update to "Paying Customer" triggers commissions.
  FIX:    No fix needed. Implement the ClickBank postback (Phase 2) or
          manually update referral status via /api/track/status.

PROBLEM: Purchase events don't create commissions (after ClickBank setup)
  CAUSE:  (a) The affid was not passed via cvendthru to ClickBank
          (b) The affid in the postback is invalid or affiliate is inactive
          (c) customer_email was not provided (needed for referral matching)
  FIX:    Ensure the cvendthru parameter is appended to ClickBank URLs,
          the affid is valid, and customer_email is included.

PROBLEM: Console shows 404 for /api/track/event
  CAUSE:  This was a bug in earlier deployments — the endpoint was missing.
  FIX:    This has been fixed. The endpoint now exists. If you still see
          404s, clear your browser cache and reload the page.

PROBLEM: Old tracker JS is being served (cache issue)
  CAUSE:  Browser or CDN is caching the old version of the tracker JS.
  FIX:    Hard-refresh the page (Ctrl+Shift+R) or add a cache-buster:
          src="https://aff-massapro.space-z.ai/massapro-affiliate-tracker.js?v=3"
